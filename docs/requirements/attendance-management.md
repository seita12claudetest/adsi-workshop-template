# 勤怠管理アプリ — 要求仕様書

## 概要

社内従業員の勤怠管理を行う Web アプリケーション。出退勤・休憩の打刻、有給・休暇申請、残業管理、承認ワークフローを提供する。

## ユーザーと規模

- 利用者数: 1,000人以上
- 組織構造: 本部 → 部 → 課（3階層）
- 拠点: 3箇所

## ロール

| ロール | 説明 |
|--------|------|
| 一般社員 | 自身の打刻、申請、勤怠確認 |
| 上長 | 部下の勤怠承認・差戻（課の長が自動的に上長） |
| 管理者（ADMIN） | 全操作（マスタ管理、全社員の勤怠操作、拠点管理等） |

## 勤務体系

| 項目 | 内容 |
|------|------|
| 勤務形態 | 固定時間制 |
| 勤務時間帯 | 9:15 〜 17:30 |
| 休憩時間 | 12:00 〜 13:00（1時間固定） |
| 所定労働時間 | 7時間15分/日 |
| 残業計算 | 日の所定労働時間（7:15）超過分 |

## 機能一覧

### F-001: 出退勤打刻

- **ユーザーストーリー**: 一般社員として、Web画面のボタンで出勤・退勤を打刻したい。勤務実績を正確に記録するために。
- 位置情報（GPS）を打刻時に取得・記録する
- 許可エリア外の場合は警告を表示するが、打刻は可能とする
- 打刻種別: 出勤 / 退勤

### F-002: 休憩打刻

- **ユーザーストーリー**: 一般社員として、休憩の開始・終了を打刻したい。実際の休憩時間を記録するために。
- 打刻種別: 休憩開始 / 休憩終了
- 所定休憩（12:00〜13:00）との差異を確認できる

### F-003: 勤務時間自動計算

- **ユーザーストーリー**: 一般社員として、日次・月次の勤務時間を自動計算してほしい。手計算の手間を省くために。
- 実労働時間 = 退勤 - 出勤 - 休憩時間
- 残業時間 = 実労働時間 - 所定労働時間（7:15）
- 月次集計: 総労働時間、総残業時間、出勤日数

### F-004: 月次勤怠一覧

- **ユーザーストーリー**: 一般社員として、自分の月次勤怠一覧を確認したい。勤務実績を振り返るために。
- カレンダー形式で日別の出退勤・休憩・労働時間を表示
- 月の合計（労働時間、残業時間、有給取得日数）を表示

### F-005: 有給・休暇申請

- **ユーザーストーリー**: 一般社員として、有給休暇や特別休暇を申請したい。事前に休暇の承認を得るために。
- 対象休暇種別:
  - 年次有給休暇（全日）
  - 半休（午前 / 午後）
  - 時間有給
  - 特別休暇: 慶弔休暇、育児休暇、介護休暇、生理休暇
- 申請時に理由（任意）と期間を入力
- 残日数を超える申請はエラーとする

### F-006: 有給残日数管理

- **ユーザーストーリー**: 一般社員として、自分の有給残日数を確認したい。計画的に休暇を取得するために。
- 付与ルール: 一斉付与日（4/1）に全社員へ法定日数を付与
- 消化: 承認済み休暇取得時に自動で減算
- 残日数: 付与 - 消化 をリアルタイム表示
- 繰越: 前年度未消化分の繰越（法定2年）

### F-007: 残業申請・承認

- **ユーザーストーリー**: 一般社員として、残業を事前申請したい。上長の承認を得てから残業するために。
- 事前申請: 予定残業時間と理由を入力
- 事後申請: 打刻ベースで所定超過があった場合に申請可能

### F-008: 打刻修正申請

- **ユーザーストーリー**: 一般社員として、打刻忘れ・誤打刻を修正申請したい。正確な勤怠記録を保つために。
- 本人が修正内容を入力して申請
- 上長が承認すると打刻が修正される

### F-009: 承認ワークフロー

- **ユーザーストーリー**: 上長として、部下の各種申請を承認・差戻したい。チームの勤怠を適切に管理するために。
- 対象: 有給申請、残業申請、打刻修正申請、勤怠月次確定
- 承認階層: 1段階（直属の上長のみ）
- 操作: 承認 / 差戻（差戻時はコメント必須）
- 管理者は全社員の申請を承認・差戻可能

### F-010: 通知機能

- **ユーザーストーリー**: 利用者として、重要なイベントの通知を受け取りたい。対応漏れを防ぐために。
- 通知種別:
  - 打刻忘れリマインド（退勤未打刻の場合、当日夕方に通知）
  - 申請の承認依頼通知（上長向け）
  - 申請の承認・差戻結果通知（申請者向け）
  - 有給残日数アラート（残日数が一定以下になったとき）
- 通知方法: アプリ内通知（画面上のベル通知）

### F-011: 位置情報・拠点管理

- **ユーザーストーリー**: 管理者として、許可拠点を登録・管理したい。打刻場所の妥当性を確認するために。
- 拠点情報: 名称、住所、緯度・経度、許可半径
- 3拠点を初期登録、管理者が追加・編集・削除可能
- 打刻時に最寄り拠点との距離を判定

### F-012: 社員マスタ管理

- **ユーザーストーリー**: 管理者として、社員情報を管理したい。組織変更や入退社に対応するために。
- 管理項目: 社員ID、氏名、メールアドレス、所属（本部/部/課）、ロール、入社日
- 課の長を指定 → その課の社員の上長になる

### F-013: 組織マスタ管理

- **ユーザーストーリー**: 管理者として、組織構造を管理したい。部署の新設・統廃合に対応するために。
- 3階層: 本部 → 部 → 課
- 各階層の作成・編集・削除

## 認証・セキュリティ

- 認証方式: ID（社員ID or メールアドレス）+ パスワード
- パスワードはハッシュ化して保存（BCrypt）
- セッション管理: JWT トークン
- ロールベースのアクセス制御

## 技術スタック

| レイヤー | 技術 |
|----------|------|
| Backend | Java / Spring Boot |
| Frontend | TypeScript / Next.js |
| DB | PostgreSQL |
| インフラ | AWS (ECS + RDS) |
| 認証 | Spring Security + JWT |

## 非機能要件

| 項目 | 要件 |
|------|------|
| 同時接続数 | 1,000人以上を想定 |
| レスポンスタイム | 画面表示 2秒以内 |
| 可用性 | 営業時間帯（8:00〜22:00）は稼働 |
| データ保持 | 勤怠データは法定期間（5年）保持 |
