# Unit 5: 有給管理

## Phase: C（Unit 1, 4 の一部完了後）

## 目的

有給休暇の付与・消化・残高管理を提供する。一斉付与（4/1）と繰越（2年有効）に対応する。

## ユーザーストーリー

- 一般社員として、自分の有給残日数を確認したい（F-006）
- 管理者として、4/1 に全社員へ有給を一斉付与したい（F-006）
- システムとして、休暇申請承認時に残日数を自動で減算したい（F-006）
- システムとして、残日数を超える休暇申請をエラーにしたい（F-005）

## スコープ

### Entity

- LeaveBalance（有給残高）

### Service

- LeaveBalanceService:
  - 残高照会（有効な残高一覧、合計残日数）
  - 有給付与（一斉付与バッチ）
  - 有給消化（承認時に自動減算）
  - 残日数チェック（申請時のバリデーション）
  - 失効処理（expiry_date 超過分の無効化）

### 付与ロジック

- 付与日: 毎年 4/1
- 付与日数: 勤続年数に基づく法定日数
  - 6ヶ月: 10日
  - 1年6ヶ月: 11日
  - 2年6ヶ月: 12日
  - 3年6ヶ月: 14日
  - 4年6ヶ月: 16日
  - 5年6ヶ月: 18日
  - 6年6ヶ月以上: 20日
- 失効: 付与日から2年（例: 2026/4/1 付与 → 2028/3/31 失効）
- 消化順: 古い付与分から消化（FIFO）
