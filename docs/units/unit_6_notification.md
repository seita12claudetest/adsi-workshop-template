# Unit 6: 通知

## Phase: C（Unit 4, 5 完了後）

## 目的

アプリ内通知（ベル通知）の生成・配信・既読管理を提供する。

## ユーザーストーリー

- 利用者として、打刻忘れのリマインドを受け取りたい（F-010）
- 上長として、承認依頼の通知を受け取りたい（F-010）
- 申請者として、承認・差戻の結果通知を受け取りたい（F-010）
- 利用者として、有給残日数が少なくなったらアラートを受け取りたい（F-010）
- 利用者として、通知を既読にしたい

## スコープ

### Entity

- Notification（通知）

### Service

- NotificationService:
  - 通知生成（各イベント発生時に呼び出される）
  - 通知一覧取得（未読優先、ページネーション）
  - 未読数取得
  - 既読化（個別 / 全件）

### 通知トリガー

| トリガー | 通知種別 | 受信者 | タイミング |
|---------|---------|--------|-----------|
| 退勤未打刻 | CLOCK_REMINDER | 本人 | 当日 17:30 以降（バッチ） |
| 申請作成 | APPROVAL_REQUEST | 上長 | 申請時（即時） |
| 承認・差戻 | APPROVAL_RESULT | 申請者 | 承認/差戻時（即時） |
| 有給残 ≤ 3日 | LEAVE_BALANCE_ALERT | 本人 | 消化時（即時） |

### Repository

- NotificationRepository

## テーブル

- notifications

## API

| メソッド | パス | 説明 |
|----------|------|------|
| GET | /api/v1/notifications | 通知一覧 |
| GET | /api/v1/notifications/unread-count | 未読数 |
| PUT | /api/v1/notifications/{id}/read | 既読にする |
| PUT | /api/v1/notifications/read-all | 全て既読 |
